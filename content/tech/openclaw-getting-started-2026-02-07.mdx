## 为什么 OpenClaw 值得从“零到一”跑通一次
OpenClaw 的核心价值不是“更聪明的聊天”，而是**把你的消息变成可执行的行动**。官方文档把目标定义为：从零到第一条可工作的对话，只需要最小配置即可跑通。核心依赖只有两件事：**Node 22+** 与 **Gateway**。

![OpenClaw 快速上手流程](/images/openclaw-getting-started.png)

---

## 第 1 步：准备环境（官方前置）
官方 Getting Started 要求非常明确：
- **Node 22 或更高版本**
- 具备基本 CLI 环境

这一步的关键不是“装什么”，而是**保证你能在本机或服务器上运行一个长期服务进程**。OpenClaw 的 Gateway 是常驻进程，后续所有通道与会话都依赖它。

---

## 第 2 步：启动 Gateway（系统的中枢）
OpenClaw 的设计是“一个 Gateway 连接所有渠道”。官方说明：
- Gateway 是**会话、路由与频道连接**的唯一入口
- 默认配置即可启动
- 控制面板可通过 `http://127.0.0.1:18789/` 访问

**建议：**如果你打算长期使用，最好在服务器上以 service 的方式运行（避免重启后中断）。

---

## 第 3 步：连接消息渠道
官方文档强调：OpenClaw 是“**多渠道统一入口**”，WhatsApp、Telegram、Discord、iMessage 等都可以共用一个 Gateway。

关键点：
- **路由是确定性的**，消息来自哪里，就回到哪里
- **SessionKey 决定上下文**（DM 与群组会话分离）

这一步完成后，才算真正把“助手”接到你的日常沟通入口。

---

## 第 4 步：验证首条对话
官方 Quick Start 的终点是“**第一条可用对话**”。这一步应验证：
- 能否收到你的消息
- 能否正确回复
- 工具调用/记忆是否正常

**一个实用检查：**
发送“请总结当前时间并写到日志”，看是否能调起工具并返回结果。

---

## 关键环境变量（官方推荐）
官方文档给出三类常用变量：
- `OPENCLAW_HOME`：改变内部路径解析
- `OPENCLAW_STATE_DIR`：会话与状态目录
- `OPENCLAW_CONFIG_PATH`：配置文件路径

当你在服务器上部署时，这三项可以让配置与数据**完全可控**。

---

## 小结：最小可用 = Gateway + 频道 + 首次对话
OpenClaw 的“从 0 到 1”不是复杂的安装，而是**跑通整个消息 → 路由 → 回应闭环**。只要第一条对话成功，后面的能力（cron、webhook、自动化）才有意义。
